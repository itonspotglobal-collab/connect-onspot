Fix the Talent ProfileOnboarding modal so that saving a profile persists both profile data and skills selections into the database.

Requirements:

1. Frontend:
   - ProfileOnboarding form must collect:
     • first_name, last_name, title, bio, location, hourly_rate, rate_currency, availability, profile_picture, phone_number, languages, timezone.
     • skills (array of skill IDs or names).
   - On Save/Submit Profile:
     • Call authAPI.post("/profiles", profileData) for profile fields.
     • Call authAPI.post("/profiles/skills", { skills }) to save skills for the same user_id.
   - Ensure Authorization Bearer token is attached automatically via axios interceptor.
   - After save:
     • Show success toast: "Profile saved successfully!".
     • Refresh AuthContext or refetch profile and skills so user immediately sees updates.
   - On error:
     • Show toast with backend error message.

2. Backend:
   - Implement POST /api/profiles:
     • Require Authorization.
     • Verify JWT with process.env.JWT_SECRET.
     • Use req.user.id as profiles.user_id.
     • If profile exists → UPDATE it.
     • Else → INSERT a new row.
     • Return { success:true, profile }.

   - Implement POST /api/profiles/skills:
     • Require Authorization.
     • Verify JWT.
     • Use req.user.id to manage user_skills.
     • Remove existing user_skills for this user_id.
     • Insert new rows for each selected skill { user_id, skill_id, level="intermediate", years_experience=0 }.
     • Return { success:true, skills }.

   - Implement GET /api/profiles/me:
     • Return profile + associated skills for the logged-in user.

3. Database:
   - Confirm mapping:
     • profiles table for main profile info.
     • user_skills table for skills selection (with FK user_id → users.id).
   - Ensure constraints (unique user_id) are respected.

4. Testing:
   - Development:
     • Create a Talent.
     • Open ProfileOnboarding, set Hourly Rate=30, select “C#” as a skill, save.
     • Verify `profiles.hourly_rate = 30`.
     • Verify `user_skills` contains a row with that user_id and C# skill_id.
   - Update profile again with Hourly Rate=40 and add “Accounting”.
     • Confirm profile row updated and user_skills reflects exactly new selection.
   - Production: repeat tests, ensure no duplicate rows.

Deliverable:
A working Talent ProfileOnboarding flow where Save Profile stores or updates the user’s profile in `profiles`, updates skills in `user_skills`, and provides clear success/error messages.
